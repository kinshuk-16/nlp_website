What is it about?

“When Congress passes laws, federal agencies implement those laws through regulations. These regulations vary in subject, but include everything from ensuring water is safe to drink to setting health care standards. Regulations.gov is the place where users can find and comment on regulations.” - Regulation.gov

Currently, all the comments are analyzed by a group of individuals. While this is sufficient when the volume of comments is low, the process fails when a particular regulation receives a large volume of comments. In our project, we aim to automate the process of summarizing and clustering comments on government regulations in order to aid policymakers in making better decisions based on the comments as well as to allow other interested parties to explore the documents and the comments being submitted.

To do this, we plan to first extract important key phrases from the text of the regulation and use them as categories for clustering the comments. As the categories and key phrases will vary for each regulation, we will use unsupervised methods to develop them. While the methods will be unsupervised, we will hand tag various regulations and the associated comments for model development and testing purposes. After determining the appropriate categories, we will analyse the text in each comment and match it with the category from the regulation text. Additionally, we will attempt to go a step further and also try to extract the overall sentiment from each comment. Finally, we plan to host the functionality on a web application which will allow users to access the regulations of their choosing along with the gist of all of its comments.

How does it work?

Document keyword extraction
This algorithm extracts the keywords in a document. This helps the tool in giving users a high level idea of the topics that the document is talking about, and also acts as an input to other algorithms.
Key sentences from document
These are the sentences from the document that have the highest importance score, based on three criteria: highest number of our important keywords; longest length (indicating an abundance of information included); and presence of words indicating the sentence is related to the problem the regulation is attempting to solve.
Cluster comments based on topic
This algorithm clusters the comments on a document using k-means. The number of clusters is determined by the number of common parents for the top unigrams in the main document. The output for this algorithm is a collection of comment clusters based on the topic
Pull out central comment cluster sentence
As it is hard to determine the semantic meaning behind a cluster, we identified and selected the comment closest to the k-means centroid as a representative of that cluster.
Differentiating cluster phrases
Having clustered the comments, we merged all of the comments in each cluster into one ‘mega-comment’ and then used TF-IDF across these ‘mega-comments.’
